# 브랜치 전략 실습

## 사용 전략: GitHub Flow

### 브랜치 목록
- main: 항상 배포 가능한 상태 유지
- feature/add-strategy-notes: 브랜치 전략 메모 추가 작업

## 브랜치 네이밍 규칙
- feature/기능명 : 새 기능 개발
- fix/버그명     : 버그 수정
- hotfix/이슈명  : 긴급 패치

## 느낀 점
git flow나 github flow나 뭐 익숙하던건데, trunk-based는 확실히 처음 듣네.

우리 학생들한테는 git flow랑 github flow만 설명해야겠다.

실습 내용 보니 이건 merge될 때 ff로 되겠군.
